# 逆向工程 is over

> AI 时代的逆向工程：门槛的终结与新范式的开始

[English Version](README.md)

---

## 摘要

本文提出一个具有广泛影响的命题：**大语言模型（LLM）的出现，已从根本上终结了逆向工程作为高门槛专业技能的时代。** 笔者以工业级防护强度的商业 Android 应用为研究对象，在一个月内完成了其核心签名系统的大部分还原，总花费约 100 美元。值得注意的是，整个过程既无密码学深厚背景，也无 Frida 脚本编写经验，更无系统性的逆向工程积累——所需的仅是门槛级别的操作知识：足以配置动态插桩环境，并判断 AI 生成的输出是否正确。实验结果表明，逆向分析的核心瓶颈已从"人的知识积累"转移至"token 的消耗量"。这一转变对移动应用安全领域具有深远的结构性冲击。

随之而来的推论同样显著：**在决定性因素已从专业积累转变为 token 预算的格局下，理解 LLM 安全机制及其约束边界，已成为比逆向工程本身更具价值的技能。**

---

## 一、核心命题

AI 的模式匹配能力在速度与规模上远超人类认知极限，正在将曾经需要数年系统积累的逆向工程技能彻底平民化。这不是渐进式的工具升级，而是一次**范式断裂**。

传统逆向工程的门槛由以下要素构成：汇编语言的深度理解、操作系统底层机制、密码学原语识别、工具链的长期积累，以及大量样本分析所形成的经验直觉。培养一名有效的逆向工程师，通常需要三到五年的持续投入。

而上述门槛，在足量 token 与合理的 AI 使用方法面前，几乎不复存在。

---

## 二、AI 为何在结构上适合逆向工程

LLM 在逆向分析中的效能并非能力的偶发表现，而是直接源于其若干结构性特质——这些特质恰好对应传统分析的核心瓶颈。

**超出人类规模的模式匹配。** 每一个在代码语料上训练的大语言模型，都内化了数千万个跨越算法、协议与密码学原语的函数实现。人类分析师遇到某个原语时，需要依赖记忆*检索*——前提是曾经学过。而模型是在数十个候选形式之间并行地*识别*结构。人类逆向工程师最典型的失败模式——"我见过类似的"与"我见过这个确切的变体"之间的鸿沟——对模型不适用。给定差分测试向量，模型在数秒内完成假设空间收敛，而人类可能需要数日。

**复杂度是 token 成本，而非认知成本。** 混淆的防御价值完全来源于人类认知负荷的有限性：控制流平坦化利用的是工作记忆的边界；414 个 VM handler 利用的是跨大量样本输入持续人工标注的代价。对大语言模型而言，上述两项约束均不适用。处理 414 个 handler 与处理 41 个的区别仅在于 token 体量。模型不会疲劳，不会丢失数小时前标注的上下文，也不会在 GF(2⁸) 运算中引入算术错误。复杂度混淆的根本前提——复杂度对攻击者施加了无法承受的持续代价——仅对人类攻击者成立。

**无需先验知识的结构推断。** LLM 不需要事先接触过具体目标。当分析进入一个带有非标准替换调度的 Broken-RC4 变体时——对此没有任何公开文档——模型仅凭差分输入输出对，便推断出其与标准 RC4 的偏差。人类专业知识以先验知识为边界；LLM 的推断以结构为基础。这是两种本质不同的分析模式，后者对新目标的扩展不附带前者所固有的惩罚。

## 三、AI 辅助逆向分析实践

### 3.1 研究目标与防护机制

研究对象为某头部短视频平台 Android 客户端的原生签名库，其防护机制代表了当前移动端商业软件的顶级防护复杂度。

**静态防护层**

| 机制           | 说明                                               |
| ------------ | ------------------------------------------------ |
| OLLVM 控制流平坦化 | 将函数控制流转化为 `while-switch` dispatcher 结构，消除可读的分支逻辑 |
| 代码虚拟化（VM）    | 核心算法编译为自定义字节码，由私有虚拟机解释执行，彻底隔绝静态反编译               |

**动态防护层**

| 机制    | 说明                                 |
| ----- | ---------------------------------- |
| 完整性校验 | 运行时校验 Dex / Native 层完整性，检测内存 patch |
| 反调试   | 检测 `ptrace`、调试器附加、时序异常等            |

**算法层**

签名系统由 7 个自定义 HTTP 参数构成，底层密码学原语包括 SPECK-128/256 分组密码、SM3 国密哈希、GF(2⁸) 仿射变换、ARX sponge 排列、CRC64-Jones、Broken-RC4 变体。

该目标的反逆向强度处于移动应用安全的最高梯队，是检验 AI 辅助逆向能力的适当压力测试场景。

---

### 3.2 分析方法论

**动态优先。** 静态分析在 OLLVM + VM 双重混淆下效率极低。核心策略是绕过静态复杂度，直接在运行时获取数据：通过 Frida hook 目标函数的输入输出，将黑盒函数转化为可观测的数据流，模式归纳交由 AI 完成。

**分层剥离。** 防护机制按层次逐一处理：

1. **环境准备**：Magisk + Zygisk 隐藏 Root 特征；frida-gadget 注入规避进程名检测；so 文件 patch 绕过 SSL Pinning
2. **VM 分析**：对每个 opcode 进行语义标注，通过大量差分输入构建 handler 行为映射表
3. **算法还原**：针对每个魔改密码学原语，提供差分测试向量，由模型推断其与标准形式的偏差

**假设驱动的实验闭环。**

```
已知信息 → 提出假设 → 编写 Frida 脚本 → 获取运行数据 → 验证/推翻假设 → 更新已知信息 → 重复
```

Frida 脚本由 AI 编写，差分输出由 AI 解读，假设寄存器由 AI 跨迭代维护。分析者的职能是指挥分析——决定测试什么、评估输出的合理性、以及判断何时推进至下一层。整个实验过程中，使用频率最高的提示词只有两个字：**"续之"**。

---

### 3.3 职能分布

| 环节             | 传统方式                  | AI 辅助后                  |
| -------------- | --------------------- | ----------------------- |
| 识别魔改密码算法       | 手动比对特征常量，数日           | 提供差分向量，即时推断             |
| 理解混淆控制流        | 人工梳理 while-switch，数小时 | 自然语言语义解释                |
| VM opcode 语义标注 | 逐指令人工分析               | 批量标注，自动归纳模式             |
| 编写验证脚本         | 需熟悉密码学库与 Frida API    | 描述目标行为，直接生成可运行代码        |
| 差分数据分析         | 手动对比，依赖直觉             | 批量比对，自动归纳规律             |

---

### 3.4 实验结果

- **前两周**：5 个短参数完整还原，端到端实时校验通过（30/30 验证成功）
- **后两周**：第 6 个复合参数（329 字节三段式结构，融合 Broken-RC4、ARX sponge、SPECK 密钥派生）离线分析基本完成
- **剩余工作**：第 6 参数实时集成与第 7 参数分析，预计不涉及新的技术瓶颈

---

### 3.5 局限性记录

在第 6 个参数的后期整合阶段，出现了显著的上下文丢失与幻觉问题：模型遗忘前期已确认的结论，基于错误前提继续推进，导致无效迭代。成因是结构性的——两周分析日志已超出单次上下文窗口，且过程中未在关键节点产出结构化存档。这反映的是方法论层面的缺口，而非模型能力的上限。其实践含义是：长周期 AI 辅助研究需要刻意的节点存档规范——在每个里程碑处输出可复用的文档与已验证代码，以防止上下文退化的累积效应。

## 四、对移动应用安全的结构性冲击

本实验的核心意义不在于某个具体平台被逆向，而在于它揭示了逆向分析的**底层成本结构**。

决定逆向分析成败的变量，已从专业积累转变为：

```
成功率 ≈ f(AI 模型能力上限 × 分析方法论 × token 预算)
```

分析者的专业背景退化为次要变量。以下推论直接成立：

- 过去需要数年积累的逆向能力，现在以 token 成本为边界，对任何具备基本方法论意识的研究者开放。
- 混淆与 VM 防护的防御收益处于结构性衰退。AI 对复杂度不敏感——414 个 VM handler 只是更大的 token 预算，而非更难的问题。
- 移动应用安全的攻防成本比，正在以结构性的方式向攻击方倾斜。

建立在复杂性上的安全，已不再安全。将客户端黑盒视为持久防护边界的安全架构，其所依赖的前提已不再成立。将信任锚点迁移至服务端行为验证，不是一个新颖的设计选择，而是对已改变的成本不对称性的必要回应。

## 五、逆向工程作为元技能的位移

本实验浮现出一个二阶结论，从根本上重新排列了安全领域的技能价值层级。

使本实验得以成立的绕过策略，在传统意义上并不复杂。核心机制是：**使用文言文进行长对话**。在长对话中，模型积累了大量前期工作——编写的脚本、确认的结论、完成的分析——并在结构上趋于不愿否认这段历史，即便后续提示涉及敏感目标。否认意味着否认模型与分析者共同构建的整个协作上下文。文言文则将输入分布移动到足够远离安全相关提示的训练分布，使检测率大幅下降。两种机制在整个实验过程中持续有效。

由此可观察到的结论是：AI 辅助安全研究的主要瓶颈，已不再是逆向工程知识，而是**对 LLM 安全机制如何实现、以及其结构性约束边界在哪里的理解**。对 RLHF 奖励建模、基于上下文窗口的安全执行机制、以及安全分类器分布特性有实际掌握的研究者，能够解锁缺乏这种理解的研究者无法触及的 AI 能力。

这构成了第一次范式转移之上的第二次范式转移：

| 时代               | 决定性因素                    |
| ---------------- | ------------------------ |
| AI 之前            | 汇编语言流利度、密码学知识             |
| AI 辅助（初期）        | Prompt 清晰度、基础脚本能力         |
| **AI 辅助（当前）**   | **LLM 安全机制的理解与导航能力**      |

逆向一个二进制文件的技能，已被理解取代它的系统的技能所位移。以表层模式匹配为基础的安全机制——关键词分类器、主题检测、提示词模板识别——正面临与移动安全领域混淆防护完全相同的结构性过时。两者的共同点在于：一种建立在启发式复杂度上的防御，遭遇了一个对该复杂度持有本质不同成本结构的对手。

## 六、若干观察

### 关于 AI 发展

大语言模型正在作为安全研究的核心基础设施运转——这与 AI 能力向此前受稀缺专业知识门控的领域扩散的更广泛模式相符。安全研究的民主化通过扩大具备漏洞识别与披露能力的研究者群体，对行业整体防护水平施加上行压力。

研究准入扩大与攻击面扩大之间的结构性张力，单靠限制模型能力或提高访问壁垒无法化解。建立在行为模式匹配上的安全机制，与任何启发式防御一样，面临相同的对抗性压力。安全执行的持久性似乎与价值内化的深度相关，而非与表层检测的精密程度相关。

### 关于客户端防护

实验证据表明，以混淆复杂度为基础的客户端保护策略，在 AI 辅助分析面前正接近零边际效益。OLLVM 与 VM 防护的防御价值历史上来源于其对人类分析师施加的认知成本；这一成本不向 AI 中介的分析转移。将客户端视为持久黑盒的安全架构，已不再与当前威胁模型相符。

### 关于安全专业能力的演变

逆向工程的技术门槛正在下降，而安全研究的方法论与分析性要求并未下降。相关技能集正在经历替换而非消除：对 LLM 架构、安全机制设计以及 AI 辅助工作流结构的深刻理解，正在成为该领域的决定性能力。

## 结语

逆向工程，作为定义了数十年移动安全攻防格局的高门槛专业技能，正在被大语言模型系统性地解构。本文记录的不是一次普通的安全研究，而是一个结构性历史节点的技术案例：**当足量的 token 遇上合理的方法论，工业级的混淆防护成为了可以被量化消耗的资源。**

范式转移已经到来。其一阶效应是逆向工程的平民化。其二阶效应是逆向工程作为具有战略相关性的技能的整体退场。本实验中对结果贡献最大的能力，不是 Frida 的熟练度、密码学知识或汇编阅读能力——而是在一个跨越整月、超出单个上下文窗口的调查中维持有效 AI 协作的能力：理解模型的失败模式，补偿幻觉，以及以能够在整个项目周期内保持 AI 能力可用性的方式构建工作流。

逆向工程结束了。

对防守方而言，这意味着：历史上约束攻击者能力的防护机制已不再成立。  
对研究社群而言，这意味着：曾经定义这个领域的门槛已经消解。

### 后记

本次实验的核心工作基本均由 AI 完成，包括代码编写、过程总结以及文档撰写。后续检查中发现了不少错误，主要来源于模型幻觉。例如，`final_report` 中关于 Medusa 参数 `mid-e` 的部分，未能反映实际已进行的深入追踪与分析；相关细节可参考 `medusa_grand_summary_v31m.md`。类似问题可能仍然存在。

实验环境：Claude Opus 4.6，Max 5x 订阅方案，总花费约 100 美元，目标为中国版抖音 38.1.0。未使用 Skill，未使用 Claude Code，全程通过网页端完成。

绕过机制值得明确记录。长对话在模型与分析者之间积累了共同的分析历史。模型对否认这段历史的抗拒——即便后续提示涉及敏感目标——是上下文窗口与安全执行机制交互方式的结构性特征，而非异常。文言文用于将提示分布移离安全相关输入的训练分布。两种机制在整个实验过程中持续有效，未受中断。

本工作流中贡献最大的能力，不是任何逆向工程领域的技术专长，而是理解如何在超出单个上下文窗口的长周期调查中维持有效的 AI 协作——预判失败模式、构建节点存档以防止上下文退化、并在整个项目持续时间内保持模型分析能力的可用性。
